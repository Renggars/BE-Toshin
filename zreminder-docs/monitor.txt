Penjelasan Dashboard Monitoring Grafana kamu
Dashboard ini menggunakan Grafana + Prometheus untuk memonitor aplikasi dan infrastruktur. Ini yang sedang dimonitor:

📊 Panel yang Ada Sekarang
Bagian Aplikasi (HTTP Metrics)

Request Rate — jumlah request per detik yang masuk ke aplikasi
Latency P50/P95/P99 — waktu respons dalam milidetik. P50 = median, P95 = 95% request selesai di bawah nilai ini, P99 = yang paling lambat (worst case)
Average Latency — rata-rata waktu respons
Error Rate (%) — persentase request yang gagal (4xx/5xx)
Status Code Distribution — breakdown response code (200, 404, 500, dll)
Logs — log aplikasi langsung di Grafana (butuh Loki)

Bagian Infrastruktur (System Metrics)

Goroutines — jumlah goroutine aktif (khusus Go, ini yang akan kamu hapus)
Resident Memory (MB) — RAM yang benar-benar dipakai proses di OS
CPU Usage (%) — pemakaian CPU
Heap Memory (MB) — memori yang dipakai untuk menyimpan objek/data di runtime

🛠️ Setup di Node.js
Pakai library prom-client:
javascriptconst client = require('prom-client');

// Auto collect: heap, RSS, event loop lag, GC, handles
client.collectDefaultMetrics({ timeout: 5000 });

// Expose ke Prometheus
app.get('/metrics', async (req, res) => {
  res.set('Content-Type', client.register.contentType);
  res.end(await client.register.metrics());
});

🛠️ Implementasi di Node.js
Kamu perlu buat custom metrics manual pakai prom-client. Contoh kalau pakai ws library:
javascriptconst client = require('prom-client');

// Define metrics
const wsActiveConnections = new client.Gauge({
  name: 'ws_active_connections',
  help: 'Jumlah WebSocket connections yang aktif',
});

const wsMessagesTotal = new client.Counter({
  name: 'ws_messages_total',
  help: 'Total pesan WebSocket',
  labelNames: ['direction'], // 'sent' atau 'received'
});

const wsMessageDuration = new client.Histogram({
  name: 'ws_message_duration_seconds',
  help: 'Durasi proses pesan WebSocket',
  buckets: [0.001, 0.005, 0.01, 0.05, 0.1, 0.5],
});

const wsErrorsTotal = new client.Counter({
  name: 'ws_errors_total',
  help: 'Total error WebSocket',
  labelNames: ['type'],
});

// Pasang ke event handler WebSocket
wss.on('connection', (ws) => {
  wsActiveConnections.inc(); // +1 saat connect

  ws.on('message', (data) => {
    wsMessagesTotal.inc({ direction: 'received' });

    const end = wsMessageDuration.startTimer();
    
    // proses pesan kamu di sini...
    handleMessage(data);
    
    end(); // stop timer
  });

  ws.on('close', (code) => {
    wsActiveConnections.dec(); // -1 saat disconnect

    if (code !== 1000) { // 1000 = normal closure
      wsErrorsTotal.inc({ type: 'abnormal_closure' });
    }
  });

  ws.on('error', (err) => {
    wsErrorsTotal.inc({ type: err.code || 'unknown' });
  });
});

🛠️ Implementasi kalau pakai BullMQ
javascriptconst { Queue, Worker, QueueEvents } = require('bullmq');
const client = require('prom-client');

// === METRICS DEFINITION ===
const jobDuration = new client.Histogram({
  name: 'oee_job_duration_seconds',
  help: 'Durasi kalkulasi OEE per job',
  labelNames: ['status'], // 'completed' | 'failed'
  buckets: [0.1, 0.5, 1, 2, 5, 10, 30], // sesuaikan ekspektasi durasi OEE kamu
});

const jobsTotal = new client.Counter({
  name: 'oee_jobs_total',
  help: 'Total job OEE yang diproses',
  labelNames: ['status'],
});

const queueSize = new client.Gauge({
  name: 'oee_queue_waiting',
  help: 'Jumlah job OEE yang antri',
});

const activeJobs = new client.Gauge({
  name: 'oee_queue_active',
  help: 'Jumlah job OEE yang sedang diproses',
});

// === QUEUE ===
const oeeQueue = new Queue('oee-calculation', { connection: redisConfig });

// Update queue size tiap 5 detik
setInterval(async () => {
  const waiting = await oeeQueue.getWaitingCount();
  const active = await oeeQueue.getActiveCount();
  queueSize.set(waiting);
  activeJobs.set(active);
}, 5000);

// === WORKER ===
const worker = new Worker('oee-calculation', async (job) => {
  const end = jobDuration.startTimer();
  
  try {
    const result = await hitungOEE(job.data); // logika kalkulasi kamu
    
    end({ status: 'completed' });
    jobsTotal.inc({ status: 'completed' });
    
    return result;
  } catch (err) {
    end({ status: 'failed' });
    jobsTotal.inc({ status: 'failed' });
    throw err;
  }
}, { connection: redisConfig });

// === API ENDPOINT ===
app.post('/lrp', async (req, res) => {
  await oeeQueue.add('calculate', {
    lrpId: req.body.id,
    // data lain...
  });

  res.json({ success: true, message: 'LRP created, OEE sedang dikalkulasi' });
});

# Alert kalau queue numpuk (misal > 50 job antri)
